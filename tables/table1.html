const vs let vs var

Feature const let var
Redeclaration Not allowed Not allowed Allowed
Reassignment Not allowed Allowed Allowed
Scope Block Block Function
Hoisting Yes (not initialized) Yes (not initialized) Yes (initialized)
Temporal Dead Zone Yes Yes No

Key Differences

  • const: Cannot be reassigned, must be initialized during declaration
  • let: Can be reassigned, block-scoped
  • var: Function-scoped, hoisted with initialization

Code Examples

// const example
const PI = 3.14159;
// PI = 3.14; // This would cause an error

// let example
let counter = 0;
counter = 1; // This is allowed

// var example
var name = "John";
var name = "Doe"; // This is allowed (redeclaration)

// Block scope example
{
  let blockScoped = "I'm block scoped";
  var functionScoped = "I'm function scoped";
}
// console.log(blockScoped); // Error: blockScoped is not defined
console.log(functionScoped); // This works

Hoisting Examples

// var is hoisted and initialized with undefined
console.log(hoistedVar); // undefined
var hoistedVar = "I'm hoisted";

// let and const are hoisted but not initialized
// console.log(hoistedLet); // ReferenceError
// console.log(hoistedConst); // ReferenceError
let hoistedLet = "I'm hoisted but not initialized";
const hoistedConst = "Me too";

Scoping Differences

// Global scope
var globalVar = "I'm global";
let globalLet = "I'm also global";
const globalConst = "Me too";

function testScope() {
  // Function scope
  var functionVar = "I'm function scoped";
  let functionLet = "I'm block scoped (within function)";
  const functionConst = "Me too";
  
  if (true) {
    // Block scope
    var blockVar = "I'm still function scoped (hoisted)";
    let blockLet = "I'm block scoped";
    const blockConst = "Me too";
    
    console.log(blockVar); // Works
    console.log(blockLet); // Works
    console.log(blockConst); // Works
  }
  
  console.log(blockVar); // Works (var is function scoped)
  // console.log(blockLet); // Error: blockLet is not defined
  // console.log(blockConst); // Error: blockConst is not defined
}

testScope();

Scope Summary

  • var: Function-scoped or globally-scoped
  • let & const: Block-scoped ({ } blocks)
  • Variables declared with var are accessible throughout the function
  • Variables declared with let/const are only accessible within their block

Best Practices

Modern JavaScript Variable Declaration

  1. Use const by default for all variables
  2. Use let only when you need to reassign values
  3. Avoid using var in new code
  4. Declare variables as close as possible to their usage
  5. Initialize variables when you declare them

When to Use Each

Situation Recommended Explanation
Value won't change const Prevents accidental reassignment
Loop counters let Value needs to change each iteration
Variables in blocks let/const Block scoping prevents leaks
Legacy code var Only for maintaining older codebases

Why Avoid var?

  • Function scoping can lead to unexpected behavior
  • Hoisting can make code harder to understand
  • Redeclaration can cause bugs that are hard to track
  • let and const provide better predictability
