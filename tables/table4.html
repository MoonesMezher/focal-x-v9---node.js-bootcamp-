Array Methods Overview

Method Mutates Array Returns Description
push() Yes New length Adds elements to the end
pop() Yes Removed element Removes last element
shift() Yes Removed element Removes first element
unshift() Yes New length Adds elements to the beginning
splice() Yes Array of removed elements Adds/removes elements at index
sort() Yes Sorted array Sorts array elements
reverse() Yes Reversed array Reverses array order
map() No New array Creates new array with results of callback
filter() No New array Creates new array with elements that pass test
reduce() No Accumulator value Reduces array to single value
forEach() No* undefined Executes callback for each element
slice() No New array Returns shallow copy of portion of array
concat() No New array Returns merged arrays
find() No First matching element Returns first element that passes test
some() No Boolean Tests if at least one element passes test
every() No Boolean Tests if all elements pass test

Key Points

  • Mutating methods change the original array
  • Non-mutating methods return a new array or value without modifying the original
  • forEach() doesn't mutate the array itself, but can mutate elements if they are objects
  • Always check whether a method mutates if you need to preserve the original array
  • Chain non-mutating methods for complex operations

Mutating Array Methods

push() - Add elements to the end

const arr = [1, 2, 3];
const newLength = arr.push(4, 5);
console.log(arr); // [1, 2, 3, 4, 5]
console.log(newLength); // 5

pop() - Remove last element

const arr = [1, 2, 3];
const lastElement = arr.pop();
console.log(arr); // [1, 2]
console.log(lastElement); // 3

shift() - Remove first element

const arr = [1, 2, 3];
const firstElement = arr.shift();
console.log(arr); // [2, 3]
console.log(firstElement); // 1

unshift() - Add elements to the beginning

const arr = [2, 3];
const newLength = arr.unshift(0, 1);
console.log(arr); // [0, 1, 2, 3]
console.log(newLength); // 4

splice() - Add/remove elements at index

const arr = [1, 2, 3, 4, 5];
// Remove 2 elements starting at index 1, and add 'a', 'b'
const removed = arr.splice(1, 2, 'a', 'b');
console.log(arr); // [1, 'a', 'b', 4, 5]
console.log(removed); // [2, 3]

sort() - Sort array elements

const arr = [3, 1, 4, 1, 5];
arr.sort(); // Default sort (lexicographical)
console.log(arr); // [1, 1, 3, 4, 5]

// With compare function for numerical sort
const nums = [3, 1, 4, 1, 5];
nums.sort((a, b) => a - b);
console.log(nums); // [1, 1, 3, 4, 5]

reverse() - Reverse array order

const arr = [1, 2, 3];
arr.reverse();
console.log(arr); // [3, 2, 1]

When to Use Mutating Methods

  • When you want to modify the original array directly
  • When memory efficiency is important (no new array created)
  • For queue (shift/push) or stack (push/pop) operations
  • When working with large arrays and performance is critical

Non-Mutating Array Methods

map() - Transform array elements

const arr = [1, 2, 3];
const doubled = arr.map(x => x * 2);
console.log(arr); // [1, 2, 3] (unchanged)
console.log(doubled); // [2, 4, 6]

filter() - Filter array elements

const arr = [1, 2, 3, 4, 5];
const evens = arr.filter(x => x % 2 === 0);
console.log(arr); // [1, 2, 3, 4, 5] (unchanged)
console.log(evens); // [2, 4]

reduce() - Reduce to a single value

const arr = [1, 2, 3, 4];
const sum = arr.reduce((acc, curr) => acc + curr, 0);
console.log(arr); // [1, 2, 3, 4] (unchanged)
console.log(sum); // 10

slice() - Extract portion of array

const arr = [1, 2, 3, 4, 5];
const portion = arr.slice(1, 4);
console.log(arr); // [1, 2, 3, 4, 5] (unchanged)
console.log(portion); // [2, 3, 4]

concat() - Merge arrays

const arr1 = [1, 2];
const arr2 = [3, 4];
const merged = arr1.concat(arr2);
console.log(arr1); // [1, 2] (unchanged)
console.log(arr2); // [3, 4] (unchanged)
console.log(merged); // [1, 2, 3, 4]

find() - Find first matching element

const arr = [5, 12, 8, 130, 44];
const found = arr.find(element => element > 10);
console.log(arr); // [5, 12, 8, 130, 44] (unchanged)
console.log(found); // 12

some() - Test if at least one element passes

const arr = [1, 2, 3, 4, 5];
const hasEven = arr.some(x => x % 2 === 0);
console.log(arr); // [1, 2, 3, 4, 5] (unchanged)
console.log(hasEven); // true

every() - Test if all elements pass

const arr = [1, 2, 3, 4, 5];
const allPositive = arr.every(x => x > 0);
console.log(arr); // [1, 2, 3, 4, 5] (unchanged)
console.log(allPositive); // true

Benefits of Non-Mutating Methods

  • Preserve original data (important for state management)
  • Enable function chaining: arr.filter(...).map(...).reduce(...)
  • More predictable code behavior
  • Easier debugging since original data remains intact
  • Better alignment with functional programming principles

Array Method Examples

// Example 1: Data transformation pipeline
const products = [
    { id: 1, name: 'Laptop', price: 1000, category: 'electronics' },
    { id: 2, name: 'Phone', price: 500, category: 'electronics' },
    { id: 3, name: 'Shirt', price: 30, category: 'clothing' },
    { id: 4, name: 'Shoes', price: 120, category: 'clothing' },
    { id: 5, name: 'Book', price: 15, category: 'education' }
];

// Get electronics products, apply 10% discount, and extract names
const electronicProductNames = products
    .filter(product => product.category === 'electronics')
    .map(product => ({
        ...product,
        price: product.price * 0.9 // 10% discount
    }))
    .map(product => product.name);

console.log(electronicProductNames); // ['Laptop', 'Phone']
console.log(products); // Original array unchanged

// Example 2: Aggregating data with reduce
const orders = [
    { product: 'Laptop', quantity: 2, price: 1000 },
    { product: 'Phone', quantity: 3, price: 500 },
    { product: 'Headphones', quantity: 1, price: 100 }
];

const totalValue = orders.reduce((total, order) => {
    return total + (order.quantity * order.price);
}, 0);

console.log(totalValue); // 2*1000 + 3*500 + 1*100 = 3600

// Example 3: Finding and updating with map (non-mutating)
const users = [
    { id: 1, name: 'Alice', active: true },
    { id: 2, name: 'Bob', active: false },
    { id: 3, name: 'Charlie', active: true }
];

// Activate Bob without mutating original array
const updatedUsers = users.map(user => 
    user.id === 2 ? { ...user, active: true } : user
);

console.log(users); // Original unchanged
console.log(updatedUsers); // Bob is now active

// Example 4: Complex filtering with some and every
const teams = [
    { name: 'Team A', members: ['John', 'Jane', 'Mike'], completed: true },
    { name: 'Team B', members: ['Sarah', 'Tom'], completed: false },
    { name: 'Team C', members: ['Emily', 'David', 'Lisa'], completed: true }
];

// Check if all teams with more than 2 members are completed
const allLargeTeamsCompleted = teams
    .filter(team => team.members.length > 2)
    .every(team => team.completed);

console.log(allLargeTeamsCompleted); // true

// Example 5: Flattening arrays with flat and flatMap
const nestedArray = [1, [2, 3], [4, [5, 6]]];
const flattened = nestedArray.flat(2); // Depth of 2
console.log(flattened); // [1, 2, 3, 4, 5, 6]

// Using flatMap to map and flatten in one step
const sentences = ["Hello world", "Goodbye moon"];
const words = sentences.flatMap(sentence => sentence.split(' '));
console.log(words); // ["Hello", "world", "Goodbye", "moon"]

Common Patterns

  • Chain filter().map() for data transformation pipelines
  • Use reduce() for aggregation operations (sum, average, grouping)
  • Combine find() with spread operator for updating specific elements
  • Use some() and every() for validation checks
  • Utilize flat() and flatMap() for working with nested arrays

Best Practices

  1. Prefer non-mutating methods when working with state management
  2. Use descriptive variable names for chained operations
  3. Break complex chains into intermediate variables for debugging
  4. Always provide initial value for reduce() to avoid errors
  5. Use the second parameter of forEach/map/filter for index when needed
  6. Consider using Object.freeze() to prevent accidental mutations
