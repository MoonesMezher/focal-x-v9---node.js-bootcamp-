Function Types Comparison

Feature Function Declaration Function Expression Arrow Function
Hoisting Fully hoisted Not hoisted Not hoisted
Syntax function name() {} const name = function() {} const name = () => {}
this binding Dynamic Dynamic Lexical (from parent scope)
Arguments object Yes Yes No
Constructor Yes Yes No
Implicit return No No Yes (when without {})

Key Differences

  • Function Declaration: Hoisted, can be called before declaration
  • Function Expression: Not hoisted, must be defined before calling
  • Arrow Function: No own 'this', shorter syntax, cannot be used as constructors
  • HOF (Higher-Order Function): Functions that take other functions as arguments or return functions
  • IIFE (Immediately Invoked Function Expression): Functions that are executed immediately after definition

Function Declaration

// Function Declaration (hoisted)
function greet(name) {
    return "Hello, " + name + "!";
}

// Can be called before declaration (due to hoisting)
console.log(greet("Moones")); // "Hello, Moones!"

// Function with default parameters
function calculateArea(width, height = 10) {
    return width * height;
}

console.log(calculateArea(5)); // 50
console.log(calculateArea(5, 7)); // 35

Characteristics

  • Hoisted to the top of their scope
  • Can be called before they are defined
  • Have their own 'this' context
  • Can be used as constructors with the 'new' keyword

Function Expression

// Function Expression (not hoisted)
const greet = function(name) {
    return "Hello, " + name + "!";
};

console.log(greet("Moones")); // "Hello, Moones!"

// Cannot be called before declaration
// console.log(greet2("Moones")); // Error: greet2 is not defined
const greet2 = function(name) {
    return "Hi, " + name + "!";
};

// Named function expression
const factorial = function fac(n) {
    return n < 2 ? 1 : n * fac(n - 1);
};

console.log(factorial(5)); // 120

Characteristics

  • Not hoisted - must be defined before calling
  • Can be anonymous or named
  • Often used as callback functions
  • Can be assigned to variables or passed as arguments

Arrow Functions

// Arrow Function (not hoisted, lexical this)
const greet = (name) => {
    return "Hello, " + name + "!";
};

// Single parameter - parentheses optional
const square = x => x * x;

// Multiple parameters - parentheses required
const multiply = (a, b) => a * b;

// No parameters - parentheses required
const sayHello = () => "Hello World!";

// Returning object literal - requires parentheses
const createUser = (name, age) => ({ name, age });

console.log(greet("Moones")); // "Hello, Moones!"
console.log(square(5)); // 25
console.log(multiply(4, 7)); // 28
console.log(createUser("Moones", 30)); // {name: "Moones", age: 30}

// 'this' behavior example
const person = {
    name: "Moones",
    tasks: ["Task 1", "Task 2", "Task 3"],
    showTasks: function() {
        this.tasks.forEach(task => {
            // 'this' refers to the person object (lexical scope)
            console.log(this.name + " needs to do: " + task);
        });
    }
};

person.showTasks();

Characteristics

  • Shorter syntax than function expressions
  • No binding of 'this' - uses 'this' from enclosing scope
  • Cannot be used as constructors
  • No arguments object (use rest parameters instead)
  • Implicit return when without curly braces

Higher-Order Functions (HOF)

// Functions that accept other functions as arguments
function calculate(a, b, operation) {
    return operation(a, b);
}

const add = (a, b) => a + b;
const subtract = (a, b) => a - b;

console.log(calculate(10, 5, add)); // 15
console.log(calculate(10, 5, subtract)); // 5

// Functions that return functions
function multiplier(factor) {
    return function(x) {
        return x * factor;
    };
}

const double = multiplier(2);
const triple = multiplier(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15

// Array methods as HOFs
const numbers = [1, 2, 3, 4, 5];

// map - transforms each element
const squared = numbers.map(x => x * x);
console.log(squared); // [1, 4, 9, 16, 25]

// filter - selects elements based on condition
const evens = numbers.filter(x => x % 2 === 0);
console.log(evens); // [2, 4]

// reduce - reduces array to a single value
const sum = numbers.reduce((acc, curr) => acc + curr, 0);
console.log(sum); // 15

Characteristics

  • Accept functions as parameters
  • Return functions as results
  • Enable functional programming patterns
  • Common in JavaScript (map, filter, reduce, etc.)

Immediately Invoked Function Expressions (IIFE)

// Basic IIFE
(function() {
    console.log("IIFE executed immediately!");
})();

// IIFE with parameters
(function(name) {
    console.log("Hello, " + name + "!");
})("Moones");

// IIFE that returns a value
const result = (function(a, b) {
    return a + b;
})(5, 10);

console.log(result); // 15

// IIFE with arrow function
(() => {
    console.log("Arrow function IIFE");
})();

// Module pattern with IIFE
const calculator = (function() {
    let memory = 0;
    
    return {
        add: function(a, b) {
            return a + b;
        },
        subtract: function(a, b) {
            return a - b;
        },
        store: function(value) {
            memory = value;
        },
        recall: function() {
            return memory;
        },
        clear: function() {
            memory = 0;
        }
    };
})();

console.log(calculator.add(5, 3)); // 8
calculator.store(10);
console.log(calculator.recall()); // 10
calculator.clear();
console.log(calculator.recall()); // 0

Characteristics

  • Executed immediately after definition
  • Creates a private scope
  • Avoids polluting the global namespace
  • Often used in module patterns
  • Can be used with arrow functions

Best Practices

  1. Use function declarations for functions you need to hoist
  2. Use arrow functions for short callbacks and when you need lexical 'this'
  3. Use function expressions for callbacks and when function needs to be assigned
  4. Use IIFE to create private scopes and avoid global pollution
  5. Leverage HOFs for functional programming patterns
